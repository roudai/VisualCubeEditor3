# フィーチャー仕様: UI レイヤー（Vue 3 + Pinia）

**フィーチャーブランチ**: `003-ui-layer`
**作成日**: 2026-05-02
**ステータス**: 完了
**入力**: spec 001（ロジックレイヤー）と spec 002（描画レイヤー）の公開 API を組み合わせ、
ユーザーがブラウザ上でキューブを操作・表示・保存できる最小完結なアプリケーションを構築する

## ユーザーシナリオとテスト *（必須）*

### ユーザーストーリー 1 - キューブ状態の表示（優先度: P1）

ユーザーがブラウザでアプリを開くと、現在のキューブ状態が SVG として表示される。
状態が変わるたびに SVG が自動更新される。

**この優先度の理由**: アプリの主機能。表示なしに操作の結果が確認できない。

**独立テスト**: `CubeDisplay` コンポーネントをマウントすると、`renderSVG` が呼び出され
その戻り値 SVG が DOM に反映されることを Vitest で検証できる。

**受け入れシナリオ**:

1. **Given** アプリ起動時、**When** `CubeDisplay` がレンダリングされる、**Then** `renderSVG(cubeState)` の出力が画面に表示される
2. **Given** キューブに手順を適用後、**When** `CubeDisplay` が再レンダリングされる、**Then** 更新後の状態が SVG に反映される
3. **Given** `renderSVG` がエラーを返す場合、**When** `CubeDisplay` がレンダリングされる、**Then** 空のコンテナが表示され UI はクラッシュしない

---

### ユーザーストーリー 2 - 手順入力によるキューブ操作（優先度: P1）

ユーザーが標準記法（例: `R U R' U'`）をテキストで入力してキューブに手順を適用できる。
また、各面の個別ムーブボタンでも操作できる。

**この優先度の理由**: キューブエディタとしての中心機能。操作なしに「エディタ」が成り立たない。

**独立テスト**: `MoveInput` コンポーネントでテキスト入力後フォームを送信すると
ストアの `applySequence` が正しいムーブ列で呼び出されることを Vitest で検証できる。

**受け入れシナリオ**:

1. **Given** 有効な記法文字列（例: `R U R' U'`）、**When** フォームを送信する、**Then** ストアに手順が適用され UI が更新される
2. **Given** 無効な記法文字列（例: `XYZ`）、**When** フォームを送信する、**Then** エラーメッセージが表示され、ストア状態は変更されない
3. **Given** 個別ムーブボタン（R CW など）、**When** クリックする、**Then** 対応する単一ムーブがストアに適用される

---

### ユーザーストーリー 3 - キューブサイズ変更（優先度: P2）

ユーザーが N=2〜7 の中からキューブサイズを選択すると、solved 状態の新しいキューブにリセットされる。

**この優先度の理由**: NxN 拡張性（Constitution 原則V）をユーザーが直接体験できる機能。
基本操作（表示・手順入力）完成後に積む拡張として位置付ける。

**独立テスト**: `SizeSelector` コンポーネントで選択値を変更すると
ストアの `setSize(n)` が対応する引数で呼び出されることを Vitest で検証できる。

**受け入れシナリオ**:

1. **Given** 現在の N=3 キューブ、**When** N=5 を選択する、**Then** `setSize(5)` が呼ばれ solved な 5×5 キューブにリセットされる
2. **Given** 手順適用済みのキューブ、**When** サイズを変更する、**Then** 進行中の手順は破棄されて新しいキューブが表示される

---

### ユーザーストーリー 4 - LocalStorage による状態永続化（優先度: P2）

アプリを閉じて再度開いた際に、前回のキューブ状態が自動的に復元される。

**この優先度の理由**: Constitution 原則VI（作成状態の永続化）。
一時作業を失わないことがユーザー体験の基本要件。

**独立テスト**: `useCubePersist` コンポーザブルを含むコンポーネントをマウントし、
ストアを変更後、LocalStorage に正しく保存されることを Vitest で検証できる。
また、LocalStorage に既存データがある状態でマウントすると、ストアが復元されることも検証する。

**受け入れシナリオ**:

1. **Given** キューブ操作後、**When** `watch` が状態変化を検知する、**Then** `vce3-cube-state` キーに `serialize(state)` が保存される
2. **Given** LocalStorage に保存済み状態がある、**When** アプリを起動する（`onMounted`）、**Then** `deserialize` で状態が復元されストアに反映される
3. **Given** LocalStorage に不正 JSON がある、**When** アプリを起動する、**Then** エラーを無視し solved 状態で起動する
4. **Given** スキーマバージョン不一致（`v ≠ 1`）のデータ、**When** アプリを起動する、**Then** deserialize の `err` を検知し solved 状態にフォールバックする

---

### エッジケース

- **N=2（ポケットキューブ）**: `setSize(2)` で正しく 2×2 キューブが表示される
- **N=7（大型キューブ）**: `setSize(7)` でパフォーマンスへの影響なく表示される
- **空入力**: 空文字を送信してもエラーを表示せずストア状態は維持される
- **LocalStorage が空**: solved 状態で起動し、エラーなし
- **LocalStorage 破損データ**: フォールバックで solved 起動し、UI クラッシュなし

## 要件 *（必須）*

### 機能要件

- **FR-001**: システムは現在の `CubeState` を `renderSVG` を通じて SVG として表示しなければならない（MUST）
- **FR-002**: ユーザーは標準記法テキストを入力してキューブに手順を適用できなければならない（MUST）
- **FR-003**: ユーザーは各面（U/D/F/B/R/L）の CW/CCW ボタンで個別ムーブを適用できなければならない（MUST）
- **FR-004**: 無効な記法入力に対してエラーメッセージを表示しなければならない（MUST）。例外を投げない
- **FR-005**: ユーザーは N=2〜7 のキューブサイズを選択できなければならない（MUST）。サイズ変更は solved 状態へリセットする
- **FR-006**: キューブ状態が変化するたびに `vce3-cube-state` キーで LocalStorage へ自動保存しなければならない（MUST）
- **FR-007**: アプリ起動時に LocalStorage から状態を復元しなければならない（MUST）。不正データの場合は solved 状態にフォールバックする
- **FR-008**: 状態管理は Pinia ストアで行い、ロジックレイヤーの純粋関数を橋渡しする（MUST）。ロジックをストアに書かない
- **FR-009**: UI レイヤーは描画レイヤー（`src/render/`）の公開 API のみを呼び出すこと（MUST NOT DOM/Canvas 直接操作）
- **FR-010**: svgdom と happy-dom の競合を防ぐため、コンポーネントテストでは描画レイヤーをモックする（MUST）

### 主要エンティティ

- **CubeStore（Pinia）**: `cubeState`・`size` の状態と `applyMove`・`applySequence`・`reset`・`setSize` のアクション
- **CubeDisplay**: ストアの `cubeState` から `renderSVG` を呼び出し、結果 SVG を `v-html` で表示するコンポーネント
- **MoveInput**: テキスト入力（`parseNotation` + `applySequence`）と面ボタン（`applyMove`）を提供するコンポーネント
- **SizeSelector**: N=2〜7 の `<select>` で `setSize(n)` を呼び出すコンポーネント
- **useCubePersist**: `onMounted` で復元、`watch` で自動保存を行うコンポーザブル

## 成功基準 *（必須）*

### 測定可能なアウトカム

- **SC-001**: アプリ起動時に SVG が表示され、手順適用後に表示が更新されることがテストで確認できる
- **SC-002**: テキスト入力から手順適用までのフローが自動テストで検証できる
- **SC-003**: LocalStorage の保存・復元・フォールバックがすべて自動テストで検証できる
- **SC-004**: UI レイヤーのラインカバレッジが 90% 以上（stores/composables/components すべて）
- **SC-005**: すべての公開インターフェースが TypeScript strict mode（`vue-tsc --noEmit`）でエラーなく通過する
- **SC-006**: `pnpm dev` でブラウザ上にアプリが表示され、基本操作（手順入力・サイズ変更）が機能する

## 前提条件

- spec 001（ロジックレイヤー）完成: `createCube`・`applyMove`・`applySequence`・`parseNotation`・`serialize`・`deserialize`
- spec 002（描画レイヤー）完成: `renderSVG`（同期）が `Result<string, RenderError>` を返す
- Vue 3 / Pinia / Vite は既存 `package.json` の依存関係に含まれる
- テスト環境は Vitest + happy-dom（コンポーネントテスト）。svgdom との競合は `vi.mock` で回避
- Undo/Redo・スクランブル自動生成・キーボードショートカット・クラウド同期はスコープ外
